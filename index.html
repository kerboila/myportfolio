Master Test Plan
  1. Introduction

    This document outlines the strategy for software testing, detailing the approach, resources, schedule, and scope of intended testing activities.

  2. Test Items

    List of software products to be tested.

    Description of the features to be tested.

  3. Features to be Tested

    Detailed list of what is to be tested.

  4. Features not to be Tested

    Detailed list of what is not to be tested and reasons for not including them.

  5. Approach

    Overview of the testing approach, including the testing levels (unit, integration, system, acceptance) and types of testing to be performed (functional, performance, usability, security, etc.).

    Description of the test techniques to be used (white box, black box, grey box).

  6. Entry Criteria

    The conditions that must be met before testing can begin. This could include items like the completion of development, successful unit testing, availability of test data, and readiness of the test environment.

  7. Exit Criteria

    The conditions that must be met for testing to be considered complete. This could include items like all test cases executed, all critical bugs fixed, and approval from stakeholders.

  8. Item Pass/Fail Criteria

    Definition of the criteria to be used to judge whether a test item has passed or failed.

  9. Test Deliverables

    List of all the documents, tools, and other components to be developed and maintained in support of the testing effort (test cases, test scripts, test data, test plans, bug reports, etc.).

  10. Testing Tasks

    Detailed list of tasks to be performed as part of the testing process, and the individuals responsible for each task.

  11. Environmental Needs

    Description of the necessary hardware and software requirements.

    Description of the test data needed and how it will be managed.

  12. Responsibilities

    Description of the roles and responsibilities of each member of the test team.

  13. Schedule

    Detailed schedule of all testing activities, including milestones and deliverables.

  14. Risks and Contingencies

    Identification of potential risks and a plan for managing those risks.

    Plan outlining the steps to be taken if identified risks materialize or unforeseen issues arise.

  15. Approvals

    Process for approving the plan, including who will sign off on the plan and the completed testing.

Test Metrics

1. Escaped Defects

2. Test Coverage

Test Execution = (Number of tests already run / Total tests to be run) x 100

Requirements Coverage = (Number of requirements covered by existing tests / Total number of requirements) x 100

3. Defects per Requirement (Requirement Defect Density)

4. Test Effort (Sub metrics)

Number of tests run per (duration): Number of tests executed / Total duration

Test review rate: Number of tests reviewed / Total duration

Defect capture rate: Total defects captured / Total test run duration

Average bugs per test: Total number of bugs / Total number of tests

5. Time to test

Avg time to create tests = Total time to create tests / Total number of tests created

Avg time to run tests = Total test run time / Total number of tests run

6. Test Cost

Cost of testing per requirement: 2000/200 = 10 dollars

Cost per test hour: 2000 / number of test hours (let’s say 200) = 100 dollars

Cost per test case: 2000 / number of test cases (let’s say 50) = 40 dollars

7. Cost per bug

Cost per bug fix = Time taken to fix * the dev’s hourly rate

8. Test Execution (Daily/Weekly Status Report)

9. Defects per Release

10. Bugs Found vs Bugs Fixed

11. Defect Resolution

Defect Resolution % = (Total number of defects fixed / Total number of defects reported) x 100

12. Test Case Effectiveness

Test Case Effectiveness = (Number of bugs found / Number of test cases executed) x 100

13. Defect Leakage

Defect Leakage = (Total numbers of defects in UAT / Total number of defects found before UAT) x 100

14. Test Completion Status

% of Test Cases Executed = (Number of test cases executed / Number of test cases created) x 100

% of Test Cases Not Executed = (Number of test cases not executed / Number of test cases created) x 100

% of Test cases Passed = (Number of test cases passed / Number of test cases executed) x 100

% of Test cases Failed = (Number of test cases failed / Number of test cases executed) x 100

% of Test cases Blocked = (Number of Test cases blocked / Number of test cases executed) x 100
